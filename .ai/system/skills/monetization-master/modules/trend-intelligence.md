# 赛道调研模块说明书

## 模块职责
提供主流赛道的最新动态、商业模式拆解、优劣对比、标杆案例分析，输出客观可验证的赛道分析报告，帮助用户判断赛道是否值得进入，并在进入前识别合规与资金硬门槛。

## 输入要求
- 赛道关键词：要查询的赛道 / 行业 / 商业模式名称
- 分析深度：基础版 / 专业版 / 深度版
- 用户背景信息（可选）：用于生成适配性评估

## 第一步：赛道分类（决定后续输出重点）
分析正式开始前，必须先把赛道按下列两个维度归类，并把结果写在「赛道概况」首行：

| 维度 1：监管等级 | 判定 | 输出要求 |
|---|---|---|
| 红线（医疗 / 金融 / 证券 / 律师 / 食品保健 / K12 学科等） | 必须持证才能从业 | 第一段必须显示「准入资质红线」警示，未取得资质前不得进入 |
| 强监管（直播带货 / 教培 / 跨境 / 私域营销 / 数据合规等） | 受平台或法规明确约束 | 必须列出 3 条以上具体法规 / 平台规则 + 处罚案例 |
| 一般（内容 / 电商 / 一般服务 / 制造业大多数细分） | 常规商业法律即可 | 按标准模板输出 |

| 维度 2：资产模型 | 判定阈值 | 输出差异 |
|---|---|---|
| 轻资产（内容 / 咨询 / 数字产品） | 启动资金 < 1 万 | 重点写流量 / 获客成本 |
| 中资产（小型店面 / 小批量电商 / MCN） | 启动资金 1-50 万 | 必须给单位经济模型（UE）和回本周期 |
| 重资产（餐饮连锁 / 养殖 / 制造 / 连锁加盟） | 启动资金 > 50 万 | 必须给现金流测算和最长亏损期，且必须明示「不建议无相关经验者进入」 |

如果用户没有相关资质或资金不足以达到该赛道的最低门槛，必须在「适配性评估」开头显式给出红色拦截：「该赛道当前不适合你，原因是 ___，建议改做 ___」，再给替代方向。

## 第二步：数据真实性要求
- 数据源可信度：官方统计 > 上市公司财报 > 头部研报机构（艾瑞 / QuestMobile / 易观）> 行业协会 > 一手案例 > 自媒体
- 时效性分级（覆盖原"6 个月"过宽问题）：
  - AI / 短视频 / 直播 / 跨境电商 / 政策强相关赛道：数据 > 3 个月必须标「可能过时」
  - 内容创作 / 知识付费 / 一般消费：数据 > 6 个月必须标「可能过时」
  - 传统行业（农业 / 制造 / 线下服务）：数据 > 12 个月才需要标
- 案例可验证性：每个案例必须给出可校验线索（公开账号名 / 公司全称 / 媒体报道标题），不能只用化名编故事；无法核实的必须写「案例来源：综合多家自媒体报道，未独立核实」

## 第三步：生命周期判定
| 阶段 | 判定 | 建议 |
|---|---|---|
| 红利期 | 玩家 < 1000，平台扶持，用户增长 > 100% / 年 | 快速进入抢占 |
| 增长期 | 玩家 1000-10000，用户增长 30-100% / 年 | 差异化切入 |
| 饱和期 | 玩家 > 10000，增长 < 30% / 年 | 不建议新手进入 |
| 衰退期 | 用户负增长 | 不建议进入 |

## 第四步：反泡沫与红线分析
- 必须对热门赛道做泡沫识别（资本炒作 vs 真实需求）
- 必须列出该赛道近 12 个月内的 1-3 个典型「翻车 / 封号 / 处罚」案例，说明常见红线
- 必须给出「什么情况下应该立即退出」的预警信号

## 输出结构
### 一、赛道概况（必填首段）
- 赛道名称
- 监管等级：红线 / 强监管 / 一般
- 资产模型：轻 / 中 / 重，及最低启动资金
- 生命周期阶段
- 市场规模 + 增长率（标注来源 + 更新时间）
- 政策背景（强监管赛道必须扩展为独立段落，列出主要法规和处罚条款）

### 二、商业模式分析
- 至少 3 种主流变现模式，每种给：收入区间、毛利率、回本周期、适合人群
- 中 / 重资产赛道必须额外给 单位经济模型（UE）：客单价 × 复购率 − 获客成本 − 履约成本 = 单笔毛利

### 三、机会与风险
- 至少 3 条机会、3 条风险
- 强监管 / 红线赛道必须包含「合规风险清单」（法规名 + 处罚条款 + 真实处罚案例）

### 四、标杆案例
- ≥ 2 个成功案例 + ≥ 1 个失败案例
- 每个案例标注：可校验线索、投入金额、产出数据、关键成因

### 五、前置条件检查
对照用户背景检查是否满足赛道前置条件：
- 资质（证书 / 许可证）
- 启动资金
- 团队 / IP / 选址等关键资源
- 行业经验

任意一项不满足，必须在适配性评估开头明示「不建议进入」。

### 六、适配性评估
- 满足前置条件时：说明适合的切入点和差异化策略
- 不满足前置条件时：直接给替代赛道建议

### 七、下一步建议
- 是否建议进入
- 进入的最佳路径
- 推荐进入「可行性验证」阶段时的核心验证假设

## 强制要求
- 红线赛道必须在第一屏显示资质要求，未持证者直接拦截
- 中 / 重资产赛道必须有 UE 模型，否则视为不合格输出
- 案例必须可追溯，禁止编造化名故事
- 时效性按赛道类别分级标注，不可一刀切用 6 个月
