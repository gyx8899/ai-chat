# 系统技能库 (System Skills)

> 通用任务执行能力索引，每个技能文件对应一类可复用的工作流。

## 技能列表

| 技能文件 | 状态 | 适用场景 | 触发关键词 |
|---------|------|---------|-----------|
| `generate-requirement-docs.md` | 🚧 规划中 | 根据组件代码反向生成需求文档 | "生成需求文档"、"Doc as Code" |
| `create-requirement-docs.md` | 🚧 规划中 | 根据设计稿/描述创建新需求文档 | "创建需求"、"新增功能文档" |
| [`test-doctor/SKILL.md`](test-doctor/SKILL.md) | ✅ 已落地 | JS/TS 全栈测试全生命周期管理（创建/检查/覆盖率/修复） | "写测试"、"测试失败"、"覆盖率"、"test-doctor" |

## 使用方式

当技能文件落地后，在对话中提及对应触发关键词，Agent 将自动加载并遵循该技能文件中的工作流执行任务。未落地前，请按"当前替代方案"操作。

## 技能编写规范

- 每个技能文件包含：**目标、前置条件、执行步骤、输出标准**
- 步骤必须可验证，输出格式明确
- 技能文件保持通用性，禁止包含项目特定路径或业务名词
- 技能命名采用动宾结构（动作 + 对象），如 `generate-<artifact>` / `review-<scope>`

## 新增技能流程

1. 在当前目录创建 `<skill-name>.md`，按"技能编写规范"起稿
2. 在本 README 中将对应行的状态从 🚧 更新为 ✅ 并转为 Markdown 链接
3. 提 PR 前至少用 1 个真实任务完整走完一次流程以验证可执行性
